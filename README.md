# Markdown: Historia, utilidad y casos de uso

## 1) ¿Qué es Markdown?
Markdown es un lenguaje de marcado ligero que permite escribir texto con formato de manera sencilla y rápida, usando símbolos simples. Su objetivo es que el texto sea fácil de leer y escribir, y que luego pueda convertirse a otros formatos como HTML sin complicaciones.

## 2) Origen e historia de Markdown
- ¿Quién lo creó?
  Fue creado por John Gruber, con la colaboración de Aaron Swartz.

- ¿En qué año?
  En el año 2004.
  
- ¿Con qué propósito?
  Se creó para facilitar la escritura de texto estructurado en la web, permitiendo que el contenido fuera legible en texto plano y fácil de convertir a HTML.


## 3) ¿Para qué sirve Markdown?
1. **Dar formato a texto fácilmente**  
   Permite aplicar títulos, listas, enlaces o negritas sin usar editores complejos.
2. **Crear documentación técnica**  
   Es muy usado para manuales, guías y documentación de software.
3. **Escribir contenido para la web**  
   Se utiliza en blogs, páginas web y plataformas de desarrollo.

## 4) Casos de uso reales
- GitHub (README.md, issues y wikis)
- Documentación técnica de software
- Blogs y páginas web
- Notas personales y apuntes de estudio
- Plataformas educativas y foros

Ejemplos:
- GitHub
- Documentación técnica

## 5) Ventajas y desventajas

### Ventajas
- Fácil de aprender y usar
- Texto legible incluso sin formato
- Compatible con muchas plataformas
- No requiere software especial

### Desventajas
- Opciones de diseño limitadas
- No es ideal para documentos muy complejos
- Puede variar según la implementación
